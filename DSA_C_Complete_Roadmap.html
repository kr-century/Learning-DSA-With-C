🔬 Algorithm Types & Complexity Analysis (IIT-Level)

1. Searching Algorithms

Linear Search

Sequential search through array elements one by one.

Visual Example:
Array: [ 12 | 45 | 23 | 51 | 19 | 8 ]
Search: 51

Step 1: Check 12 ✗
Step 2: Check 45 ✗
Step 3: Check 23 ✗
Step 4: Check 51 ✓ Found!
Time: O(n)
Space: O(1)
Use Case: Unsorted arrays

Binary Search

Divide and conquer approach - reduces search space by half each iteration.

Visual Example:
Sorted Array: [ 2 | 5 | 8 | 12 | 16 | 23 | 38 | 45 | 56 | 78 ]
Search: 23

Iteration 1: Middle = 16, 23 > 16 → Right half
Iteration 2: Middle = 38, 23 < 38 → Left half
Iteration 3: Middle = 23 ✓ Found!
Time: O(log n)
Space: O(1) [Iterative]
Precondition: Array must be sorted

💡 When to Use Which Sorting Algorithm:
  • Small datasets: Insertion Sort or Bubble Sort
  • Large random data: Quick Sort (average case very fast)
  • Guaranteed O(n log n): Merge Sort or Heap Sort
  • Nearly sorted data: Insertion Sort
  • Need stability: Merge Sort or Insertion Sort
  • Limited memory: Heap Sort (in-place)

3. Graph Algorithms

Breadth First Search (BFS)

Explores vertices level by level using queue - discovers closest vertices first.

Visual Example - Graph Traversal:

      1 ← Start
  / \
 2 --- 3
/        \
4 --- 5

BFS Order (Queue-based):
Visit: 1 → 2, 3 → 4, 5
Queue progression: [1] → [2,3] → [3,4,5] → [4,5] → [5] → []
Time: O(V+E)
Space: O(V)
Use: Shortest path (unweighted)

Depth First Search (DFS)

Explores vertices deeply using stack - goes as far as possible before backtracking.

Visual Example - Same Graph:

      1 ← Start
  / \
 2 --- 3
/        \
4 --- 5

DFS Order (Stack-based):
Visit: 1 → 2 → 4 → 5 → 3
Stack progression: [1] → [2,3] → [4,3] → [5,3] → [3] → []

Dijkstra's Algorithm

Shortest path in weighted graph with non-negative weights - greedy approach.

Visual Example - Weighted Graph:

    (1)---4---(2)
   /              \
(0)                (3)
   \              /
     2        1
     |        |
     (4)---2---(5)

Shortest path from 0 to 3: 0→1→2→3 (Distance: 4)
Alternative: 0→4→5→3 (Distance: 5)
Time: O((V+E) log V) [with priority queue]
Space: O(V)
Greedy approach

Floyd-Warshall Algorithm

All pairs shortest path algorithm. Handles negative weights (except cycles).

Time: O(V³)
Space: O(V²)
Dynamic Programming

Understanding Time Complexity Growth

📈 How Complexity Grows with Input Size

Input Size | O(1) | O(log n) | O(n) | O(n log n) | O(n²) | O(2ⁿ)
──────────────────────────────────────────────────────────────
10 | 1 | 3 | 10 | 33 | 100 | 1024
100 | 1 | 7 | 100 | 664 | 10K | 2^100
1000 | 1 | 10 | 1K | 10K | 1M | 2^1000
10000 | 1 | 13 | 10K | 133K | 100M | HUGE

⚠️ Note: O(2ⁿ) becomes infeasible very quickly! Avoid unless necessary.

Fibonacci Sequence

Recurrence: F(n) = F(n-1) + F(n-2)

With Memoization: Time O(n), Space O(n)

With Tabulation: Time O(n), Space O(n)

Space Optimized: Time O(n), Space O(1)

0/1 Knapsack

Recurrence: dp[i][w] = max(dp[i-1][w], val[i] + dp[i-1][w-wt[i]])

Time: O(n*W) | Space: O(n*W)

This is pseudo-polynomial - depends on weight value

⏱️ Detailed Complexity Analysis & Examples

What is Time Complexity?

Time Complexity = How long the algorithm takes relative to input size

Notation: f(n) where n = input size

Examples:
• f(n) = 5 → O(1) - Constant time
• f(n) = 3n + 7 → O(n) - Linear
• f(n) = n² + 2n + 5 → O(n²) - Quadratic
• f(n) = 2ⁿ + n³ → O(2ⁿ) - Exponential
• f(n) = n log(n) → O(n log n) - Linearithmic

Common Complexity Examples

O(1) - Constant Time

Example: Access array element by index
int value = arr[5]; // Always 1 operation

Why constant? No matter how big the array,
accessing by index takes same time

O(n) - Linear Time

Example: Linear search
for(int i = 0; i < n; i++) {
  if(arr[i] == target) return i;
}

Why linear? Worst case: check all n elements

O(n²) - Quadratic Time

Example: Bubble sort
for(int i = 0; i < n; i++)
  for(int j = 0; j < n; j++)
    // Compare elements

Why quadratic? Nested loops = n × n operations

O(log n) - Logarithmic Time

Example: Binary search
Each iteration halves the search space

Array size: 1000000
Iterations needed: log₂(1000000) ≈ 20

Why logarithmic? Divide search space in half each time

Space Complexity Examples

O(1) Space: Bubble sort (only variables, in-place)

O(n) Space: Merge sort (needs temporary array), Linked list

O(log n) Space: Recursive binary search (call stack)

O(n²) Space: 2D array, Adjacency matrix for graph

Master's Theorem for Recurrence Relations

For T(n) = aT(n/b) + f(n):
  • If f(n) = O(n^c) where c < log_b(a) → T(n) = O(n^log_b(a))
  • If f(n) = O(n^c) where c = log_b(a) → T(n) = O(n^c log n)
  • If f(n) = O(n^c) where c > log_b(a) → T(n) = O(f(n))

Example: Merge Sort: T(n) = 2T(n/2) + n
Here a=2, b=2, f(n)=n, c=1, log₂(2)=1 → Case 2 → O(n log n)

Choosing the Right Algorithm

Searching Problem?
├─ Array unsorted? → Linear Search O(n)
└─ Array sorted? → Binary Search O(log n)

Sorting Problem?
├─ Small dataset (<50)? → Insertion Sort
├─ Need stability? → Merge Sort or Insertion Sort
├─ Large random data? → Quick Sort (usually)
├─ Need guaranteed O(n log n)? → Merge Sort or Heap Sort
└─ Memory critical? → Heap Sort (in-place)

Graph Problem?
├─ Shortest path (unweighted)? → BFS
├─ Shortest path (weighted)? → Dijkstra
├─ All pairs? → Floyd-Warshall
├─ Minimum spanning tree? → Kruskal or Prim
├─ Topological order? → DFS or Kahn
└─ Cycle detection? → DFS or Union-Find

String/Array Problem?
├─ Pattern matching? → KMP (efficient) or Naive
├─ Substring search? → Two pointers or Hashing
└─ Anagram/Count? → Hashing or Sorting

Decision Tree for Data Structure

Need to store items in order?
→ Array (fast access, fixed size)
→ Linked List (flexible size, slower access)
→ Vector/ArrayList (dynamic array)
Need LIFO access pattern?
→ Stack (Last In, First Out)
Common uses: Function calls, Undo/Redo, Expression evaluation
Need FIFO access pattern?
→ Queue (First In, First Out)
Common uses: BFS, Task scheduling, Printer queue
Need fast search & insertion?
→ Hash Table (average O(1))
→ Binary Search Tree (worst O(log n))
→ Balanced BST (guaranteed O(log n))
Need hierarchical relationship?
→ Tree (parent-child relationship)
→ Graph (general relationships, can have cycles)

📋 Quick Reference Guide

Algorithm Properties Quick Lookup

✓ STABLE Sorting Algorithms (Maintain relative order of equal elements)
  • Bubble Sort
  • Insertion Sort
  • Merge Sort
  • Counting Sort
✗ UNSTABLE Sorting Algorithms (Don't maintain order)
  • Selection Sort
  • Quick Sort
  • Heap Sort
IN-PLACE Algorithms (Use O(1) extra space)
  • Bubble Sort - O(1)
  • Selection Sort - O(1)
  • Insertion Sort - O(1)
  • Heap Sort - O(1)
  • Quick Sort - O(log n)
NOT IN-PLACE (Need extra space)
  • Merge Sort - O(n)
  • Counting Sort - O(k)

Traversal Methods Comparison

Data Structure Iterative Traversal Recursive Traversal BFS
Array/List Simple loop Recursion + index N/A
Linked List Use pointer, iterate Recursion + next ptr Queue needed
Tree Stack or Queue Natural with DFS Queue for level-order
Graph Stack for DFS DFS visits nodes Queue for BFS

When to Use Each Data Structure

Fast Access by Index? → Array
Frequent Insertions/Deletions at Front? → Linked List
Need LIFO (Last In First Out)? → Stack
Need FIFO (First In First Out)? → Queue
Need Hierarchical Data? → Tree
Need Fast Lookup? → Hash Table / BST
Need Priority Access? → Heap / Priority Queue
Need All Relationships? → Graph

💡 Essential Tips & Tricks for DSA

📌 For Learning Efficiently:
  • Understand, don't memorize: Focus on understanding algorithms, not just memorizing code
  • Practice regularly: 2-3 hours daily practice is better than 10 hours once a week
  • Code from scratch: Try implementing without looking at solutions first
  • Use debugging tools: Learn GDB for C to debug efficiently
  • Visualize algorithms: Use tools like VisuAlgo to understand how algorithms work
  • Time yourself: Practice solving problems under time constraints
🎯 Problem-Solving Approach:
  • Read carefully: Understand the problem completely before coding
  • Think of edge cases: Empty array, single element, negative numbers, etc.
  • Start with brute force: Write a simple solution first, then optimize
  • Analyze complexity: Always think about time and space complexity
  • Test thoroughly: Test with multiple test cases including edge cases
  • Optimize gradually: Don't jump to complex optimizations
💻 Coding Best Practices:
  • Variable naming: Use meaningful variable names (not a, b, c)
  • Comments: Comment complex logic, not obvious code
  • Memory management: Free allocated memory properly in C
  • Handle errors: Check for NULL pointers, malloc failures
  • Use modular approach: Write helper functions for reusable code
  • Keep functions small: Each function should do one thing well
🧠 Interview Preparation Tips:
  • Communicate: Explain your approach before coding
  • Ask clarifying questions: Never assume about the problem
  • Start simple: Begin with a straightforward approach
  • Optimize step-by-step: Show how you improve the solution
  • Test your code: Walk through examples while explaining
  • Discuss trade-offs: Explain why you chose one approach over another
⏱️ Time Management in Exams:
  • Read all questions: First read all questions before starting
  • Allocate time: Spend more time on higher marks questions
  • Code carefully: Write code once, get it right
  • Don't skip: Answer theory questions even if you can't code
  • Complexity analysis: Always include time & space analysis
  • Draw diagrams: For data structure questions, draw diagrams
